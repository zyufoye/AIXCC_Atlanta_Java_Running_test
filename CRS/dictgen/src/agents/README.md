`agents` 文件夹下的这些文件构成了字典生成器的核心代理系统：

## 代理系统架构概述

这是一个**多代理协作系统**，每个代理负责字典生成流程中的特定任务。

## 各文件详细作用

### 1. `BaseAgent.py` - 基础代理类
**作用**：所有代理的基类，定义统一的接口和通用功能

**可能包含：**
- 代理生命周期管理
- 消息传递机制
- 错误处理
- 日志记录
- 与其他代理的协作接口

### 2. `ConstantExtractor.py` - 常量提取代理
**作用**：从源代码中提取常量值

**具体功能：**
- 识别代码中的常量定义（字符串、数字、枚举等）
- 提取可能用于字典的固定值
- 过滤无关的常量（如版本号、配置值）
- 生成常量候选列表

**示例提取：**
```java
// 提取这些常量
public static final String DELIMITER = ":";
private static final int MAX_SIZE = 1024;
String[] TYPES = {"json", "xml", "csv"};
```

### 3. `DictAnalyzer.py` - 字典分析代理
**作用**：分析和优化生成的字典

**具体功能：**
- 合并重复的字典条目
- 评估字典条目的质量
- 过滤无效或低质量的条目
- 生成最终的字典格式
- 统计字典覆盖率

### 4. `ParsingFunctionDetector.py` - 解析函数检测代理
**作用**：识别代码中的解析相关函数

**具体功能：**
- 检测字符串解析函数
- 识别输入处理函数
- 找到数据转换函数
- 标记可能产生字典条目的函数

**示例检测：**
```java
// 检测这些函数
parseJSON(), split(), tokenize(), 
decode(), deserialize(), fromString()
```

### 5. `TokenExtractor.py` - 标记提取代理
**作用**：从代码中提取有意义的标记（tokens）

**具体功能：**
- 识别代码中的关键标记
- 提取函数名、变量名中的语义信息
- 分析字符串字面量中的模式
- 生成标记候选列表

**示例提取：**
```java
// 从这些代码中提取标记
user.getName() → ["user", "name"]
parseXML(data) → ["parse", "xml"]
config.getValue("timeout") → ["config", "value", "timeout"]
```

### 6. `TriggerVerifier.py` - 触发验证代理
**作用**：验证字典条目是否能正确触发目标代码路径

**具体功能：**
- 测试生成的字典条目
- 验证条目是否能到达目标函数
- 检查条目的有效性
- 过滤无法触发目标的无用条目

## 代理协作流程

```
源代码
    ↓
ParsingFunctionDetector → 识别解析函数
    ↓
TokenExtractor → 提取语义标记
    ↓  
ConstantExtractor → 提取常量值
    ↓
DictAnalyzer → 分析和优化字典
    ↓
TriggerVerifier → 验证字典有效性
    ↓
最终字典输出
```

## 在字典生成系统中的角色

这个代理系统实现了**智能的字典生成流程**：

1. **目标识别**：找到需要生成字典的函数
2. **信息提取**：从代码中提取相关标记和常量
3. **字典构建**：基于提取的信息构建字典
4. **质量验证**：确保字典的有效性和覆盖率

## 设计优势

- **模块化**：每个代理专注单一职责
- **可扩展**：易于添加新的分析代理
- **协作性**：代理之间可以共享信息和结果
- **可测试**：每个代理可以独立测试

这个代理系统使得字典生成不再是简单的字符串提取，而是基于代码语义的智能分析过程。
